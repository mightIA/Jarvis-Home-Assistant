# Style personnel

Fichier source **canonique** : `Ressources/Competences/Traduction_Style_Personnel.md`.
Cet atome sert de **pointeur + formules de politesse figées** pour le mode
Personnel de la skill `traduction`. Enrichissement progressif à chaque
correction validée par Mickael.

## Formules de politesse — emails professionnels

| Langue | Ouverture | Clôture |
|--------|-----------|---------|
| FR | `Bonjour [Prénom],` | `Cordialement, Mickael Rubino` |
| EN | `Hello [First name],` | `Best regards, Mickael Rubino` |
| DE | `Guten Tag [Vorname],` | `Mit freundlichen Grüßen, Mickael Rubino` |

## Formules de politesse — messages personnels / familiers

| Langue | Ouverture | Clôture |
|--------|-----------|---------|
| FR | `Salut [Prénom],` | `À bientôt !` |
| EN | `Hi [First name],` | `Cheers` / `Take care` |
| DE | `Hallo [Vorname],` | `Bis bald` / `Liebe Grüße` |

## Signature selon contexte

- **Formel** : `Mickael Rubino`.
- **Décontracté** : `Mickael`.
- **Nickname / dispatch** : `Might` (rare, à valider selon contexte).

## Sections à enrichir

Le fichier source suit un format d'apprentissage progressif :

1. **Préférences globales** — langue principale FR, niveaux EN / DE à
   confirmer.
2. **Tournures préférées** — format type : règle + contexte + exemple
   AVANT (neutre) / APRÈS (style Mickael).
3. **Mots / tournures à éviter** — à enrichir au fil des corrections.
4. **Notes d'apprentissage** — processus de formalisation après correction.

## Workflow d'apprentissage

À chaque session où Mickael corrige une traduction :

1. **Formaliser** la règle observée (description + exemple avant / après).
2. **Ranger** dans la bonne section (tournure préférée, mot à éviter,
   politesse, etc.).
3. **Proposer** à Mickael la validation **avant** d'écrire dans le fichier
   source.
